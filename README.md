## 🌟 Executive Overview

**RICIS III** (*Recursive Indexed Calculus of Identity and Singularity*) is a self-consistent, axiomatically minimal, and globally continuous extension of classical mathematics. By preserving absolute identity ($L_1: X = X$) through recursive algebraic indexing, RICIS-III eliminates limits ($\lim_{x \to a}$) and indeterminate forms ($0/0$, $\infty/\infty$, $0 \cdot \infty$, $\infty - \infty$) at physical, mathematical, and computational singularities.

This repository contains the web application, symbolic calculator, 2D/3D visual interactive simulators, relational analytics database engine, and an autonomous Gemini AI expert agent for RICIS-III.

## 📐 Mathematical Foundations & Axiomatics

### Core Laws
* **$L_0$ (Absolute Continuity):** No recursion level or spatial transform permits loss of identity or topological discontinuities.
* **$L_1$ (Law of Identity):** $X = X$ and $X/X = 1$ globally. Type equals Identity ($T(X)$ preserved under morphisms).

### Safety Protocols (Singularity Resolution Rules)
1. **SP1 (Locality Rule):** In $0/0$ occurrences, $1$ replaces *only* identical zero-factors. The remaining functional "tail" stays active.
2. **SP2 (Reduction Priority):** Algebraic factorization and simplification MUST precede singularity axiom application.
3. **SP3 (Index Law):** When cancellation is impossible, $0_F / 0_G = F/G$. Treating typed zeros as scalar zeros is prohibited.
4. **SP4 (Semantic Priority):** Singularities are indexed by their generating function $E(x)$ at $x=a$, preserving algebraic structure.

### Singular Axioms
* **A1 (Indexing):** $F/0 \to \infty_F$ (Indexed Infinity).
* **A2 (Indexed Infinity Existence):** $\infty_F$ exists $\forall F \neq 0$ (with $\infty_0 = 1$).
* **A4 ($0/0$ Resolution):** $0_F / 0_G = F/G$.
* **A5 ($\infty/\infty$ Resolution):** $\infty_F / \infty_G = F/G$.
* **A6 (Product Resolution):** $0_F \times \infty_G = F \cdot G$ (for all $F, G$).
* **A7 (Difference Resolution):** $\infty_F - \infty_G = \infty_{F-G}$.
* **A10 (Zero Typing):** $F \times 0 = 0_F$.

## 🗄️ Database Architecture & Smart Visitor Tracking

The backend utilizes SQLite (`node:sqlite` DatabaseSync) in WAL mode with normalized relational schemas:

```
[ users ] (user_key PRIMARY KEY)
    │
    ├──< [ user_ip_links ] >──┐
    │                         │
[ visit_logs ]         [ ip_addresses ] (ip PRIMARY KEY)
    │                         │
    └─────────────────────────┘
```

### Smart Recognition Pipeline
1. **Multi-Source Token Identification:** Client identification checks HTTP request header `X-Visitor-Token`, browser HTTP cookie `ricis_visitor_token`, and `localStorage` token.
2. **Automatic Token Provisioning:** New visitors are automatically issued a persistent `usr_*` token via HTTP headers and cookies on their first request.
3. **Cross-Device IP Linkage:** Maps multiple IP addresses to a single user key across devices while maintaining a complete history in `user_ip_links` and `visit_logs`.
4. **Server Log Ingester (`scanServerLogsAndSeedDatabase`):** Automatically scans system web logs (`/var/log/nginx/access.log`, etc.) on initialization to seed IP addresses and access statistics if the database is fresh.

## 🛠️ Installation & Local Development

### Prerequisites
* Node.js v20.x or higher
* npm or bun

### Setup
```bash
# 1. Clone repository
git clone https://github.com/dimaaley/ricis-iii-calculator.git
cd ricis-iii-calculator

# 2. Install dependencies
npm install

# 3. Configure environment variables (optional for Gemini AI Agent)
cp .env.example .env
# Add GEMINI_API_KEY in .env if desired

# 4. Start Development Server
npm run dev
# Server will run on http://localhost:3000

# 5. Build for Production
npm run build
npm start
```
